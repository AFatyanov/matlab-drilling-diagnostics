# MATLAB Drilling Diagnostics Tool

Профессиональный MATLAB-инструмент для моделирования и диагностики осложнений при бурении нефтегазовых скважин.

## Что создано

### Структура проекта
```
matlab_diagnostics/
├── main_drilling_diagnostics.m          (62 строки) - главный скрипт запуска
├── generate_synthetic_drilling.m        (158 строк) - генерация синтетических данных
├── inject_complications.m               (109 строк) - моделирование осложнений
├── calculate_diagnostic_features.m      (117 строк) - расчёт диагностических признаков
├── detect_events_level1.m               (110 строк) - детектор уровня 1 (эвристики)
├── detect_events_level2.m               (108 строк) - детектор уровня 2 (инженерные расчёты)
├── detect_events_level3.m               (101 строка) - детектор уровня 3 (статистика)
├── aggregate_detections.m               (126 строк) - агрегация результатов
├── plot_diagnostics.m                   (107 строк) - построение графиков
├── generate_report.m                    (113 строк) - генерация текстового отчёта
├── save_results.m                       (128 строк) - экспорт результатов
└── output/                              (создаётся автоматически при запуске)
```

**Всего: 1239 строк чистого MATLAB кода в 11 файлах**

## Что получается на выходе

После запуска создаётся папка `output/` с 8 файлами:

1. **raw_drilling_data.csv** - синтетические временные ряды (480 точек × 15 параметров)
2. **diagnostic_features.csv** - рассчитанные инженерные и статистические признаки
3. **detected_events.csv** - таблица обнаруженных осложнений с деталями
4. **drilling_diagnostics_results.mat** - полный MATLAB workspace
5. **diagnostic_summary.txt** - текстовый отчёт на русском языке
6. **diagnostic_timeseries.png** - график временных рядов с выделением осложнений
7. **risk_timeline.png** - график изменения risk score
8. **events_for_web.json** - JSON для интеграции с web-интерфейсом

## Характеристики данных

- **Период симуляции**: 5 суток непрерывного бурения
- **Шаг дискретизации**: 15 минут
- **Количество точек**: 480
- **Параметры**: MD, TVD, ROP, WOB, RPM, Torque, Hookload, SPP, Flow Rate, Mud Weight, ECD, Pit Volume, Gas, Pore Pressure, Fracture Gradient

## Смоделированные осложнения

1. **Kick** (газонефтеводопроявление) - точки 150-180
2. **Mud Losses** (поглощение) - точки 250-270
3. **Pack-off** (затяжки/посадки) - точки 330-360
4. **Stuck Pipe** (прихват колонны) - точки 420-450

## Архитектура диагностики

### Трёхуровневая система обнаружения:

**Уровень 1: Эвристики**
- Простые пороговые правила
- Быстрое обнаружение очевидных признаков
- Confidence: 0.65-0.75

**Уровень 2: Инженерные расчёты**
- ECD vs Pore Pressure и Fracture Gradient
- Анализ трендов давления, момента, ROP
- Комбинированные правила
- Confidence: 0.75-0.85

**Уровень 3: Статистика**
- Скользящие окна, базовая линия
- Z-score анализ
- Скорость изменения параметров
- Раннее обнаружение аномалий
- Confidence: 0.60-0.70

### Агрегация результатов:
- Объединение событий от трёх детекторов
- Расчёт итогового risk score (0-100)
- Определение confidence level (низкий/средний/высокий)
- Классификация severity (minor/moderate/severe/critical)
- Формирование инженерного объяснения на русском языке

## Ограничения модели

1. Синтетические данные не учитывают все реальные факторы
2. Пороговые значения подобраны эмпирически
3. Возможны ложные срабатывания при соединениях свечей
4. Не учтены геологические осложнения
5. Отсутствует ML для адаптации под конкретные условия
6. Детекторы настроены на обнаружение, а не прогнозирование
